# Security Controls Checklist (Portal + Lab Data)

Administrative

- [ ] DPO designated and active.
- [ ] Privacy training completed for staff handling patient data.
- [ ] Access rights approved by role.
- [ ] Incident response SOP documented and tested.

Technical

- [ ] HTTPS enforced in production.
- [ ] CAPTCHA enabled on portal login.
- [ ] Consent required before verify/report.
- [ ] Rate limiting enabled.
- [ ] Service account key stored server-side only.
- [ ] Least privilege access on Google resources.
- [ ] Audit logging enabled (consent and access events).

Physical/Operational

- [ ] Workstations protected with account-level access.
- [ ] Shared drives/folders permission reviewed monthly.
- [ ] Disposal procedure for printed records documented.
